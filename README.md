# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 야구 선수 기록 관리 프로그램
<br>



## 2. 시나리오 (5~10줄)
* 이 프로그램은 야구 선수들의 기록을 체계적으로 관리하는 '야구 선수 기록 관리 시스템'입니다.
  
* 가장 먼저 프로그램을 실행하면 사용자는 새로운 선수의 이름과 타수, 안타와 같은 기본 기록을 입력하게 되며 이 데이터를 바탕으로 선수 관리가 시작됩니다. 입력된 기록을 통해 선수의 타율을 계산하고 이를 통해 해당 선수의 현재 경기력을 확인할 수 있습니다. 특히 일정 기준 이상의 높은 타율을 기록한 선수에게는 '우수 선수' 라는 등급을 부여하여 성과를 강조할 수 있습니다. 이후에는 한 명의 선수만 관리하는 것이 아니라 여러 선수들의 데이터를 리스트에 저장하여 한 번에 관리할 수 있도록 확장할 예정입니다. 사용자는 저장된 선수 목록을 확인하고 각 선수의 기록을 비교하며 팀 전체의 전력을 파악할 수 있습니다. 최종적으로는 메뉴 기반 시스템을 통해 선수 추가, 조회, 수정 등의 기능을 자유롭게 선택할 수 있는 야구 선수 기록 관리 프로그램으로 발전시키는 것이 목표입니다. 또한 선수 데이터를 기반으로 팀 전력 분석 기능까지 확장하는 것이 마지막 목표입니다.
<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 신입 선수 등록
2. 타율 계산
3. 선수 성적 등급 판정
4. 우수 선수 보상 및 칭호 부여
5. 선수 목록 전체 출력
0. 프로그램 종료

    * **적용 내용:** AI의 답변을 통해 '야구팀 기록 분석가' 컨셉이 프로그램의 구조와 가장 잘 맞는다고 판단하였다. 선수의 타수, 안타, 타율과 같은 데이터를 입력하고 분석하는 과정이 자연스럽게 연결되기 때문에 해당 컨셉을 채택하였다. 이를 통해 단순한 기록 프로그램이 아니라 선수의 성적을 평가하고 분석하는 프로그램으로 방향을 구체화할 수 있었다.
      
 2. **내용 2 : 기능 확장 로드맵 도출**
    * **프롬프트 요약:** "야구 선수 기록 관리 프로그램을 1차부터 4차 과제까지 입출력, 리스트, 메뉴, 함수 구조로 확장할 떄 각 단계별로 어떤 기능을 추가하면 좋을지 질문함."
    * **적용 내용:** AI와의 논의를 통해 프로그램의 단계별 개발 방향을 구체적으로 설계하였다. 1차에서는 선수 정보 입력과 타율 계산 기능을 구현하고 2차에서는 리스트를 활용하여 여러 선수 데이터를 관리하도록 계획하였다. 3차에서는 메뉴 기반 반복 구조를 도입하여 기능 선택이 가능하도록 하고 4차에서는 함수와 딕셔너리를 활용하여 프로그램을 체계적으로 구조화하는 방향으로 설정하였다. 이 과정을 통해서 프로그램의 전체적인 확장 흐름을 명확하게 정리할 수 있었다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 이번 과제는 야구 선수 기록 관리 시스템을 확장하여 사용자로부터 여러 선수의 데이터를 입력받아 리스트에 저장하고 조건문을 활용하여 선수의 성적을 분석하는 프로그램을 구현하는 것이다.
  * input()을 사용하여 선수의 이름, 타수, 안타를 입력받고 리스트에 저장한다.
  * 저장된 데이터를 기반으로 타율을 계산한다. (타율 = 안타 / 타수)
  * if-elif-else 조건문을 활용하여 타율에 따른 등급(S, A, B, C, F)을 판정하는 기능을 구현한다.
  * 추가적으로 특정 기준 이상의 타율을 가진 선수에게 '우수 타자' 칭호를 부여하는 기능을 구현한다.
    
### **🤖 AI 파트너십 과정**
1. **내용 1: 야구 분석가 컨셉 확장 및 로직 도출**
    * **프롬프트 요약:** "야구 선수 기록 관리 프로그램에서 리스트와 조건문을 활용하여 여러 선수의 데이터를 관리하고 성적을 분석하는 방법을 질문함."
    * **적용 내용:** AI의 답변을 통해 단일 선수 데이터가 아닌 여러 선수의 정보를 리스트에 저장하는 방식으로 확장하였다.
또한 타율을 기준으로 조건문을 적용하여 선수의 성적을 S, A, B, C, F 등급으로 나누는 로직을 설계하였다.
이를 통해 프로그램이 단순 계산을 넘어 선수 성적을 분석하는 구조로 발전하였다.

2. **내용 2: 데이터 구조 및 조건문 설계**
    * **프롬프트 요약:** "리스트를 활용하여 선수 데이터를 저장할 때 어떤 구조가 효율적인지와, 타율 기준 등급을 나누는 조건문 설계를 질문함."
    * **적용 내용:** AI와의 논의를 통해 선수 정보를 리스트 형태로 저장하고, 각 선수의 타수와 안타 데이터를 기반으로 타율을 계산하도록 설계하였다.
이후 if-elif-else 조건문을 활용하여 타율 구간별 등급을 판정하고, 특정 기준 이상일 경우 추가 칭호를 부여하는 기능을 구현하였다.
이를 통해 데이터 처리와 조건 판단이 결합된 프로그램 구조를 완성할 수 있었다.
    
### **🛠️ Troubleshooting & 기술 회고:**
1. **문제 1: 타율 계산 시 오류 발생 (ZeroDivisionError)**
    * **원인:** 타수를 0으로 입력했을 때 안타 ÷ 타수 계산에서 0으로 나누게 되어 오류가 발생함.
    * **해결:** 타수가 0일 경우 타율을 0으로 처리하거나 계산을 하지 않도록 조건문을 추가하여 문제를 해결함.
2. **문제 2: 조건문이 원하는 대로 작동하지 않는 문제**
    * **원인:** if-elif 조건 순서를 잘못 설정하여 높은 타율이 낮은 등급으로 분류되는 문제가 발생함.
    * **해결:** 조건을 높은 값부터 순서대로 비교하도록 수정하여 올바른 등급이 출력되도록 개선함.
